PUT THE RELEASE KEYSTORE IN THIS FOLDER (mobile/)
==================================================

WHAT
    The production signing key for the sideloaded Android APK. Every release
    must be signed with the SAME key -- Android refuses to install an update
    signed by a different one, and officers would have to uninstall and lose
    local state to recover.

    Convention (what this project's setup expects):

        mobile/firenet-release.keystore
        key alias: firenet

    The path isn't hardcoded -- it comes from a Gradle property (see below),
    so any location works. Keeping it here is just convention.

WHERE TO GET IT
    The keystore already exists -- this project has shipped releases. It lives
    on the production server, which holds a full clone at ~/www/firenet:

        scp <your-ssh-user>@wildfire.forest.go.th:~/www/firenet/mobile/firenet-release.keystore \
            mobile/firenet-release.keystore

    The passwords are deliberately NOT stored on the server next to it. Ask a
    maintainer for those separately.

    Do NOT generate a replacement. A new keystore is a different identity and
    permanently breaks updates for every installed device. The section below
    is for bootstrapping a brand-new deployment, not for this one.

    Local development doesn't need this file at all -- debug and dev-client
    builds sign themselves.

HOW TO CREATE ONE (brand-new deployment only -- not this project)
    Once per deployment, ever. Never to "fix" an existing one: if any device
    has a build signed with a different key, a new keystore orphans it. Unsure
    whether one exists? Ask a maintainer first.

    Needs a JDK -- `keytool` ships with the same one that builds the app.

        keytool -genkeypair -v \
          -keystore firenet-release.keystore \
          -alias firenet \
          -keyalg RSA -keysize 2048 -validity 10000 \
          -dname "CN=FireNET, OU=wildfire, O=<your org>, L=<city>, ST=<state>, C=<country code>"

    It prompts for a keystore password and a key password (twice each, no
    echo); using the same value for both is fine. The ~27-year validity is
    deliberate -- this key must outlive every device it ever signed for.

    You get one file holding the private key and a self-signed certificate.
    Nothing is registered with Google or Firebase; it's a local identity.

    Back up that file AND both passwords somewhere durable and access-
    controlled before going further. Losing them is unrecoverable.

HOW TO INSTALL IT
    1. Put the .keystore file here (or anywhere you prefer).
    2. Add these to your GLOBAL Gradle config at ~/.gradle/gradle.properties
       -- NOT to any file inside this repo:

           FIRENET_UPLOAD_STORE_FILE=<absolute-path-to-your-clone>/mobile/firenet-release.keystore
           FIRENET_UPLOAD_STORE_PASSWORD=<ask a maintainer, or what you set above>
           FIRENET_UPLOAD_KEY_ALIAS=firenet
           FIRENET_UPLOAD_KEY_PASSWORD=<ask a maintainer, or what you set above>

       The path is YOUR machine's -- e.g. a clone at C:\Projects\FireNET-RFD
       gives C:/Projects/FireNET-RFD/mobile/firenet-release.keystore. Forward
       slashes even on Windows (Gradle treats \ as an escape). Point it at a
       file that doesn't exist and assembleRelease fails loudly at signing.

    Consumed by mobile/plugins/withReleaseSigning.js, which rewrites the
    generated android/app/build.gradle at prebuild time.

WARNING -- THE FAILURE MODE HERE IS SILENT
    If the properties are ABSENT entirely, the release build does not fail --
    it falls back to the DEBUG keystore. The APK builds, publishes, and then
    refuses to install over any real release. publish-apk.ps1 checks the
    version code but not the signing key, so nothing catches it but officers.

    Always verify before shipping:

        apksigner verify --print-certs <path-to.apk>

    The certificate must match previous releases, not "Android Debug".

DO NOT COMMIT IT
    Already covered by mobile/.gitignore (`*.keystore`, `*.jks`,
    `keystore.properties`). Losing or leaking this key is unrecoverable.

LEAVE THIS .TXT IN PLACE
    It is tracked in git as a marker for the next person setting up. Deleting it
    locally just shows up as a pending deletion in your working tree.
